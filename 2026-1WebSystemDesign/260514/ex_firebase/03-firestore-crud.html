Note12-03 · Firestore CRUD 4함수

슬라이드: CRUD 4함수 — addDoc / getDocs / updateDoc / deleteDoc

📌 이 데모는 한 번 읽기(getDocs)만 사용. 실시간 자동 갱신은 다음 데모(04). 추가/수정/삭제 후 "다시 읽기"를 눌러야 화면 갱신됨.

코드 핵심

// Create
await addDoc(collection(db, 'memos'), {
  text, authorId: auth.currentUser?.uid, createdAt: serverTimestamp()
});

// Read (한 번)
const snap = await getDocs(collection(db, 'memos'));
const memos = snap.docs.map(d => ({ id: d.id, ...d.data() }));

// Update
await updateDoc(doc(db, 'memos', id), { text: '수정된 내용' });

// Delete
await deleteDoc(doc(db, 'memos', id));

자주 헷갈리는 점

  • getDoc (단수, 1개) vs getDocs (복수, 컬렉션)
  • 추가/수정/삭제 후 화면 안 바뀜 → 이 데모는 한 번 읽기. 자동 갱신은 04 데모(onSnapshot)
  • serverTimestamp() — 클라이언트 시간 X, 서버 시간 사용 (시계 조작 방지)
  • permission-denied → 보안 규칙이 막음 → 05 데모에서 다룸
