Note12-07a · 이미지 첨부 (base64 + Firestore)

슬라이드: Firebase Storage 무료 대안 — Spark 요금제에서도 동작하는 가장 단순한 길

📌 방식: 이미지를 클라이언트에서 Canvas로 리사이즈 + JPEG 압축 → base64 dataURL로 변환 → Firestore 문서에 통째로 저장. 별도 서비스 가입 없음, Storage 활성화 불필요. 썸네일·아바타·작은 첨부에 적합.

⚠️ 한계: Firestore 문서 최대 1MB. 이 데모는 자동으로 max 400×400 / quality 0.7로 리사이즈해서 보통 30~80KB로 떨어뜨림. 원본 이미지 보관이 필요하면 07b-image-cloudinary.html 참고.

핵심 코드

// 1) 파일 → Canvas 리사이즈 → JPEG → base64 dataURL
async function resizeToDataURL(file, maxSize = 400, quality = 0.7) {
  const img = await createImageBitmap(file);
  const scale = Math.min(maxSize / img.width, maxSize / img.height, 1);
  const canvas = new OffscreenCanvas(img.width * scale, img.height * scale);
  canvas.getContext('2d').drawImage(img, 0, 0, canvas.width, canvas.height);
  const blob = await canvas.convertToBlob({ type: 'image/jpeg', quality });
  return await new Promise(r => {
    const reader = new FileReader();
    reader.onload = () => r(reader.result);  // "data:image/jpeg;base64,..."
    reader.readAsDataURL(blob);
  });
}

// 2) Firestore에 dataURL 통째로 저장
await addDoc(collection(db, 'posts_b64'), {
  text, photoData: dataURL,  // ← dataURL을 photoData 필드에 박음
  authorId, authorName, createdAt: serverTimestamp()
});

// 3) 표시 — img src에 dataURL 그대로
<img src="${m.photoData}">

자주 막히는 곳

  • 원본 큰 사진 (4000×3000 사진 등) — 그대로 저장 시도하면 Firestore 1MB 한도 초과. 이 데모는 자동 리사이즈로 해결
  • OffscreenCanvas 미지원 브라우저 — Safari 구버전 등. 일반 <canvas> + document.createElement('canvas')로 폴백 가능
  • onSnapshot 결과 무거움 — base64가 같이 따라오니 글 100개 = 5~8MB 로딩. 글 많아지면 본문 컬렉션 / 이미지 별도 서브컬렉션 분리 또는 Cloudinary 전환 권장
  • permission-denied — Firestore 보안 규칙에 posts_b64 컬렉션 허용 추가 필요

보안 규칙 추가 (Firestore Rules 탭에 붙여넣기)

match /posts_b64/{postId} {
  allow read: if request.auth != null;
  allow create: if request.auth != null
                && request.auth.uid == request.resource.data.authorId
                && request.resource.size() < 1_000_000;  // 1MB 안전 컷
  allow delete: if request.auth.uid == resource.data.authorId;
}
